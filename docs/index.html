BrowseFleetMCP pairs a local stdio MCP server with a Chrome extension so coding agents and direct shell workflows can drive your real browser with isolated sessions.

Install Clients CLI
  1. 01 Build the root MCP server.
  2. 02 Build and load the Chrome extension.
  3. 03 Add the server to Codex, Cursor, Claude, or another MCP client.
  4. 04 Or run it directly from the command line.

Browser actions only work after a tab is connected from the extension popup. If not, tool calls will fail with No connected tab.

What you install

Server, extension, client config, or direct CLI.

Root package runs the stdio MCP server and CLI.

Chrome extension bridges tabs into leased sessions.

Client config launches the same server from Codex, Cursor, Claude Code, or generic stdio MCP hosts.

GitHub Pages

The docs site is ready for Pages, social previews, and a custom domain.

The workflow in .github/workflows/pages.yml deploys the docs/ directory directly to GitHub Pages at https://jakubjdolezal.github.io/browsermcp/.

Social preview metadata is now included on the landing page and the major subpages, using docs/assets/social-card.png generated from the source card at docs/assets/social-card.svg.

For a custom domain, copy docs/CNAME.example to docs/CNAME, replace it with your actual domain, then update the canonical and Open Graph URLs if you want the metadata to point at the custom hostname instead of the default GitHub Pages URL.
