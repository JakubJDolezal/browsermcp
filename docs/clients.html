MCP Clients

Use the same server. Change only the launcher shape.

BrowseFleetMCP is a standard stdio server. The differences between clients are mostly where the config lives and whether the client expects a command line or a config file.

Base launch pairs

{
  "command": "npx",
  "args": ["-y", "browsefleetmcp"]
}

{
  "command": "node",
  "args": ["/absolute/path/to/browsermcp/dist/index.js"]
}

Codex

Codex supports direct CLI setup with codex mcp add and manual edits to ~/.codex/config.toml. The same config is shared between the CLI and the IDE integration. 

codex mcp add browsefleet -- npx -y browsefleetmcp

[mcp_servers.browsefleet]
command = "npx"
args = ["-y", "browsefleetmcp"]

Cursor

Put the server config in ~/.cursor/mcp.json, then restart Cursor. 

{
  "mcpServers": {
    "browsefleet": {
      "command": "npx",
      "args": ["-y", "browsefleetmcp"]
    }
  }
}

Claude Code

Use the CLI for local or project-scoped setup. If you want the config checked into the repo, use --scope project. 

claude mcp add --transport stdio browsefleet -- npx -y browsefleetmcp

{
  "mcpServers": {
    "browsefleet": {
      "command": "npx",
      "args": ["-y", "browsefleetmcp"]
    }
  }
}

Generic stdio clients

Any client that can launch a local command with command and args should work with the same entrypoint. 

{
  "command": "npx",
  "args": ["-y", "browsefleetmcp"]
}

Windows note

Wrap npx with cmd /c when native Windows clients cannot spawn it directly

{
  "command": "cmd",
  "args": ["/c", "npx", "-y", "browsefleetmcp"]
}
