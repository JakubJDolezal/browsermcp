Command Line Interface

BrowseFleetMCP already ships with a direct CLI.

You can run the stdio server directly from the shell, set ports, override broker ports, or add an auth token without going through an MCP client first. 

browsefleetmcp
browsefleetmcp serve
browsefleetmcp --help
browsefleetmcp --version

What the CLI is for

  • Run the server outside any MCP client.
  • Choose preferred and fallback ports.
  • Set broker ports separately.
  • Protect the local transport with a shared token.

Direct run

Start the stdio server directly

The bare command and the explicit serve subcommand both start the same stdio MCP server. 

browsefleetmcp
browsefleetmcp serve

Local checkout

Run the current repo checkout directly

This is the best path when you are actively developing the repo and do not want your client or shell session to pull the published npm package. 

npm run build
node /absolute/path/to/browsermcp/dist/index.js

Flags

Supported CLI options

--port

Preferred browser WebSocket port.

--fallback-ports

Comma-separated backup browser WebSocket ports.

--broker-port

Preferred internal broker port.

--broker-fallback-ports

Comma-separated backup internal broker ports.

--auth-token

Shared token used to authenticate the extension and broker.

Ports

Browser and broker overrides

browsefleetmcp --port 9200 --fallback-ports 9202,9204

browsefleetmcp \
  --broker-port 9300 \
  --broker-fallback-ports 9302,9304

Auth token

Set a shared token from the CLI or the environment

browsefleetmcp --auth-token your-shared-token

BROWSEFLEETMCP_AUTH_TOKEN=your-shared-token browsefleetmcp

Inspector

Open the MCP inspector during local development

The repo ships an inspector script in package.json for local debugging. 

npm run inspector
