Local Setup

Build the server. Load the extension. Connect the first tab.

The server and extension are both required. Client config is only the launch path. Without a connected tab, the browser tools have nowhere to attach.

Client Configs CLI Flags

Install checklist

  • Build the root package in the repo root.
  • Build extension-v2/ and load it in Chrome.
  • Optionally choose ports or an auth token.
  • Connect a tab from the extension popup.

Step 1

Build the MCP server

Install dependencies and build the CLI plus stdio server bundle from the repo root. 

npm install
npm run build

Step 2

Build and load the extension

Build the Chrome extension, then load extension-v2/ in chrome://extensions with Developer mode enabled. 

cd extension-v2
npm install
npm run build

Ports

Choose your local ports if needed

By default BrowseFleetMCP prefers browser port 9150 with fallbacks 9152 and 9154. You can override both browser and broker ports from the CLI. 

browsefleetmcp --port 9200 --fallback-ports 9202,9204
browsefleetmcp --broker-port 9300 --broker-fallback-ports 9302,9304

Auth

Lock the local transport down with a shared token

If you set an auth token, enter the same token in the extension popup before connecting a tab. 

browsefleetmcp --auth-token your-shared-token

BROWSEFLEETMCP_AUTH_TOKEN=your-shared-token browsefleetmcp

Connect

Connect a tab from the popup

Popup state matters

Confirm the port settings, add the token if you are using one, and connect the current tab. If no tab is connected, browser calls fail with No connected tab.

Focus-sensitive actions

browser_click, browser_drag, browser_hover, browser_press_key, and browser_type require focus. BrowseFleetMCP serializes those actions behind a global focus lock.

Session inspection

If you connect multiple tabs, use browser_list_sessions and browser_switch_session to move the current MCP client between them.

Snapshot versus screenshot

browser_snapshot gives the simplified accessibility/navigation view. browser_screenshot captures the rendered page as it actually looks in the browser.
